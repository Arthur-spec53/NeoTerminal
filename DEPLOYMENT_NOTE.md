# 📝 部署方式说明

## 🎯 为什么只提供 Nginx 部署？

**NeoTerminal** 专注于提供最佳的部署体验，我们选择 Nginx 作为唯一推荐的部署方式，原因如下：

### ✅ Nginx 部署的优势

1. **与 XBoard 后端完美集成**
   - 已针对 XBoard 后端 API 优化
   - CORS 配置完全兼容
   - 反向代理配置开箱即用

2. **生产环境最佳实践**
   - 性能卓越，资源占用低
   - SSL/HTTPS 配置简单（自动 Let's Encrypt）
   - 支持多域名、负载均衡
   - 日志完善，便于运维

3. **一键部署脚本**
   - 从零环境到完整部署
   - 自动安装所有依赖
   - 自动配置防火墙
   - 健康检查和部署报告

4. **社区支持**
   - Nginx 文档完善
   - 问题容易排查
   - 大量生产实践

### ❌ 为什么不提供 Docker 部署？

1. **XBoard 后端未提供官方 Docker 镜像**
   - 需要用户自行构建后端镜像
   - 配置复杂度大幅增加

2. **前后端分离场景不适合**
   - 前端是纯静态文件，Nginx 足够
   - Docker 增加了不必要的复杂性
   - 资源开销更大

3. **维护成本高**
   - 需要维护多个 Dockerfile
   - Docker Compose 配置复杂
   - 数据持久化问题

4. **部署脚本已经很简单**
   - 一行命令完成部署
   - 自动处理所有依赖
   - 比 Docker 更简单

---

## 🚀 推荐部署架构

```
用户浏览器
    ↓
Nginx (前端静态文件 + 反向代理)
    ↓
XBoard 后端 (PHP + MySQL + Redis)
```

**这是最简单、最稳定、性能最好的部署方式！**

---

## 💡 如果你确实需要 Docker...

虽然我们不提供官方支持，但如果你确实需要容器化部署：

### 方案 1: 仅容器化前端

```bash
# 构建前端镜像
docker build -t xboard-frontend .

# 运行容器
docker run -d -p 80:80 xboard-frontend
```

然后手动配置 Nginx 反向代理到 XBoard 后端。

### 方案 2: 自行编写完整的 docker-compose

需要自己处理：
- ✅ MySQL 容器
- ✅ Redis 容器
- ✅ XBoard 后端容器（需自己构建）
- ✅ 前端容器
- ✅ Nginx 容器
- ✅ 数据持久化
- ✅ 网络配置

**复杂度很高，不推荐！**
